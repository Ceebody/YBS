üîß The Translation Dictionary

How E-R concepts become SQL code:

E-R Concept What it Means SQL Translation Example
Entity A "thing" to store data about CREATE TABLE Student, Product, Order
Attribute Characteristic of entity COLUMN Name, Age, Price
Primary Key Unique identifier PRIMARY KEY StudentID, ProductCode
Foreign Key Points to another table FOREIGN KEY CourseID in Student table
Relationship Connection between entities JOIN operations Student ENROLLS_IN Course

üèóÔ∏è Building a Complete Database: Step by Step

Step 1: Create the Student Table
CREATE TABLE Students (
  StudentID INT PRIMARY KEY,
  Name VARCHAR(100) NOT NULL,
  Age INT,
  Email VARCHAR(150)
);
Step 2: Create the Course Table
CREATE TABLE Courses (
  CourseID INT PRIMARY KEY,
  CourseName VARCHAR(100),
  Teacher VARCHAR(100)
);
Step 3: Create the Enrollment Table (MANY-TO-MANY)
CREATE TABLE Enrollments (
  EnrollmentID INT PRIMARY KEY,
  StudentID INT,
  CourseID INT,
  Grade CHAR(1),
  FOREIGN KEY (StudentID) REFERENCES Students(StudentID),
  FOREIGN KEY (CourseID) REFERENCES Courses(CourseID)
);

üéØ Enrollments table = The BRIDGE between Students and Courses!
